## في 30 ثانية

خزّنتَ "محمد" في قاعدة بياناتك. مستخدم بحث عن نفس الاسم منسوخاً من ملف PDF: "ﻣﺤﻤﺪ". لم يجد شيئاً.

السلسلتان تبدوان متطابقتين. في يونيكود، هما **سلسلتا بايتات مختلفتان تماماً**:

    "محمد"  →  d985 d8ad d985 d8af           (8 بايتات)
    "ﻣﺤﻤﺪ"  →  efbba3 efbaa4 efbba4 efbaaa   (12 بايتة)

ضاد يعطيهما **هوية واحدة**:

```rust
use dhad::modes::process_mode_a;

let a = process_mode_a("محمد".as_bytes())?;
let b = process_mode_a("ﻣﺤﻤﺪ".as_bytes())?;

assert_eq!(a.core_hash, b.core_hash);
// كلاهما: 7661c73ca2970a2a7d6824c4e3a27560...
```

كلمة عربية واحدة. بصمة حتمية واحدة. للأبد.

هذا مُتحقَّق تجريبياً في [`PROOFS.md`](./PROOFS.md) (Proof 2).

## لماذا ضاد؟

النص العربي في يونيكود يعاني من **غموض تمثيلي حاد**:

- الحرف الواحد له حتى **5 أشكال موضعية** (معزول، بدائي، وسطي، نهائي، قانوني)
- الأرقام لها **3 تمثيلات** (ASCII، عربي هندي، عربي هندي ممدد)
- ترتيب التشكيل **قابل للتبديل** (فتحة+شدة = شدة+فتحة بصرياً)
- **أحرف غير مرئية** (ZWJ، BOM، علامات اتجاه، كشيدة) قد تتسلل بصمت

هذا يكسر:

- **البحث** — `"محمد"` لن يجد `"ﻣﺤﻤﺪ"` في قاعدة البيانات
- **الفهرسة** — نفس الكلمة مُخزّنة بـ 5+ مفاتيح مختلفة
- **كشف التكرار** — المحتوى المكرر يفلت
- **التحقق** — "هل هذه الآية مطابقة للنسخة الأم؟" تصبح بلا إجابة
- **معالجة NLP** — نماذج اللغة ترى ضوضاء بدل بنية

ضاد يحل هذا بـ **مواصفة رسمية** (23 ثابتاً، 7 تصحيحات)، **تنفيذ حتمي بـ Rust** (284 اختباراً)، و**مجموعة توافق من 185 ناقلاً** متحقق منها عبر تنفيذين مستقلين.

## البدء السريع

أضف ضاد إلى مشروعك:

```bash
cargo add dhad
```

عالج النص العربي (الأمثلة أدناه تستخدم حزمة `hex` لعرض البصمات — أضفها بـ `cargo add hex`):

```rust
use dhad::modes::process_mode_a;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let result = process_mode_a("بِسْمِ اللَّهِ".as_bytes())?;

    println!("الذرات:   {}", result.stream.len());
    println!("الإملائي: {}", hex::encode(result.core_hash));
    println!("الصوتي:   {}", hex::encode(result.phonetic_hash));

    Ok(())
}
```

استخدم سطر الأوامر (يقرأ UTF-8 من stdin):

```bash
echo -n "بسم" | dhad-cli
```

## ما تحصل عليه

سبعة ضمانات متحقَّقة، كل واحد مدعوم بإثبات تجريبي:

| الضمان | الوصف | الإثبات |
|--------|-------|---------|
| **الحتمية** (A1) | نفس المدخل يُنتج نفس المخرج، للأبد | كل الإثباتات |
| **لا تصحيح صامت** (A2) | مدخل غير صالح ← خطأ مُصنّف، لا تخمين | Proof 9 (CR-07) |
| **فصل البصمات** (A3) | البصمة الإملائية والصوتية مستقلتان | Proof 6 |
| **استقلالية الشكل** (A5) | الأشكال الموضعية لا تحمل معلومات | Proofs 1, 2 |
| **استقلالية ترتيب التشكيل** (A6) | ترتيب علامات التشكيل لا يؤثر | Proof 5 |
| **استقلالية مصدر الأرقام** (A7) | `1` = `١` = `۱` | Proofs 3, 4 |
| **تصفية الضوضاء** | BOM، ZWJ، الكشيدة تُزال بصمت | Proof 8 |

كل ادعاء قابل لإعادة الإنتاج من [`PROOFS.md`](./PROOFS.md).

## بصمتان، غرضان

ضاد يُنتج **بصمتين مستقلتين** لكل نص:

### البصمة الإملائية (CoreHash)

> "هل كُتبا بنفس الطريقة؟"

تلتقط الحروف والتشكيل والأعلام البنيوية (همزة، مدة).

استخدامها: البحث، كشف التكرار، مقارنة شكل الكتابة.

### البصمة الصوتية (PhoneticHash)

> "هل يُنطقان بنفس الطريقة؟"

تلتقط CoreHash + المعلومات العَروضية (تنوين، مد، ألف خنجرية).

استخدامها: البحث الصوتي، التحليل العَروضي، التحقق من التلاوة.

## القيود المعروفة

ضاد v1.x صادق بشأن ما **لا** يفعله بعد.

### CoreHash يشمل التشكيل

نفس الكلمة العربية مع تشكيل وبدونه تُنتج CoreHash **مختلفاً**. هذا بالتصميم (حسب المواصفة §6.1):

```rust
let unvocalized = process_mode_a("محمد".as_bytes())?;
let vocalized   = process_mode_a("مُحَمَّد".as_bytes())?;

assert_ne!(unvocalized.core_hash, vocalized.core_hash);
```

هذا يعكس الواقع اللغوي: التشكيل يحمل معلومات دلالية في العربية (`عَلِمَ` "knew" مقابل `عُلِمَ` "was known").

**للبحث الفضفاض عبر الأشكال المُشكَّلة**، جرّد التشكيل قبل استدعاء ضاد:

```rust
fn strip_diacritics(s: &str) -> String {
    s.chars()
        .filter(|c| {
            let cp = *c as u32;
            !(0x064B..=0x065F).contains(&cp) && *c != '\u{0670}'
        })
        .collect()
}

let normalized = strip_diacritics("مُحَمَّد");
let result = process_mode_a(normalized.as_bytes())?;
// يطابق الآن CoreHash لـ "محمد"
```

بصمة "الحروف فقط" قيد المناقشة عبر RFC-001 (راجع خارطة الطريق).

### ملف تقييس NFC

ضاد يتوقع النص العربي في **صيغة NFC المُركّبة مسبقاً**. علامات التركيب المُفككة (`U+0653`, `U+0654`, `U+0655`) تُنتج `UnmappedCodepoint`.

إن كان مصدرك قد يحتوي على نص مُفكك، حوّله إلى NFC قبل `process_mode_a`.

### علامات التلاوة القرآنية

علامات التلاوة الممتدة (`U+06D6`–`U+06ED`) تُنتج `UnmappedCodepoint` في v1.x. ملف `QuranicRelaxed` مخطط لـ v1.3.0.

## المساهمة

المجالات الأكثر تأثيراً:

- **غلاف Python** عبر PyO3 (`pip install dhad`)
