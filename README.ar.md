## في 30 ثانية

هذه السلاسل الثلاث تبدو متطابقة:
"محمد" "مُحَمَّد" "ﻣﺤﻤﺪ"

text


في يونيكود، هي **سلاسل بايتات مختلفة تماماً**. البحث في قاعدة بياناتك
لن يجدها. كشف التكرار لديك يتعطّل. مقارنة النصوص تفشل.

ضاد يعطيها **هوية واحدة**:

```rust
use dhad::modes::process_mode_a;

let a = process_mode_a("محمد".as_bytes())?;
let b = process_mode_a("مُحَمَّد".as_bytes())?;
let c = process_mode_a("ﻣﺤﻤﺪ".as_bytes())?;

assert_eq!(a.core_hash, b.core_hash);   // نفس الكلمة، بغض النظر عن التشكيل
assert_eq!(a.core_hash, c.core_hash);   // نفس الكلمة، بغض النظر عن الشكل
كلمة عربية واحدة. بصمة حتمية واحدة. للأبد.

لماذا ضاد؟
النص العربي في يونيكود يعاني من غموض تمثيلي حاد:

الحرف الواحد له حتى 5 أشكال موضعية (معزول، بدائي، وسطي، نهائي، قانوني)
الأرقام لها 3 تمثيلات (ASCII، عربي هندي، عربي هندي ممدد)
ترتيب التشكيل قابل للتبديل (فتحة+شدة = شدة+فتحة بصرياً)
أحرف غير مرئية (ZWJ، BOM، علامات اتجاه) قد تتسلل بصمت
هذا يكسر:

البحث — "محمد" لن يجد "مُحَمَّد"
الفهرسة — نفس الكلمة مُخزّنة بـ 5 مفاتيح مختلفة
كشف التكرار — المحتوى المكرر يفلت
التحقق — "هل هذه الآية مطابقة للنسخة الأم؟" تصبح بلا إجابة
معالجة NLP — نماذج اللغة ترى ضوضاء بدل بنية
ضاد يحل هذا بـ مواصفة رسمية (23 ثابتاً، 7 تصحيحات)، تنفيذ حتمي،
ومجموعة توافق من 185 ناقلاً متحقق منها عبر تنفيذين مستقلين.

البدء السريع
التثبيت
Bash

cargo add dhad
الاستخدام
Rust

use dhad::modes::process_mode_a;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let result = process_mode_a("بِسْمِ اللَّهِ".as_bytes())?;

    println!("الذرات:   {}", result.stream.len());
    println!("إملائي:  {}", hex::encode(result.core_hash));
    println!("صوتي:    {}", hex::encode(result.phonetic_hash));

    Ok(())
}

الميزة	الضمان
الحتمية (A1)	نفس المدخل ← نفس المخرج، للأبد، في كل تنفيذ مطابق
لا تصحيح صامت (A2)	مدخل غير صالح ← خطأ مُصنّف، لا تخمين
فصل البصمات (A3)	البصمة الإملائية والصوتية مستقلتان
استقلالية الشكل (A5)	الأشكال الموضعية لا تحمل معلومات
استقلالية ترتيب التشكيل (A6)	ترتيب علامات التشكيل لا يؤثر
استقلالية مصدر الأرقام (A7)	1 = ١ = ۱
السلامة الدلالية (A8)	التركيبات المتناقضة مرفوضة
بصمتان، غرضان
ضاد يُنتج بصمتين مستقلتين لكل نص:

البصمة الإملائية (CoreHash) — هوية الكتابة
"هل كُتبا بنفس الطريقة؟"

تلتقط الحروف والتشكيل والأعلام البنيوية (همزة، مدة).
استخدامها: البحث، كشف التكرار، مقارنة شكل الكتابة.

البصمة الصوتية (PhoneticHash) — هوية النطق
"هل يُنطقان بنفس الطريقة؟"

تلتقط CoreHash + المعلومات العَروضية (تنوين، مد، ألف خنجرية).
استخدامها: البحث الصوتي، التحليل العَروضي، التحقق من التلاوة.
